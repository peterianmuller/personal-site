These are some thoughts on the difference between the splice and slice methods that reason about arrays. Both work on arrays and begin work at the index passed as the first argument to each method. splice will remove all elements starting at the index of the first parameter, while slice will make a copy and leave the original array untouched:

var myArray = [1,2,3,4,5];
console.log(myArray.slice(2)); //[3,4,5]
console.log(myArray); [1,2,3,4,5]
console.log(myArray.splice(2)); [3,4,5]
console.log(myArray); [1,2,3,4,5];

Well if we want to work on a portion of the original array without affecting the original array, then we'd better use slice and our original array will be unaffected. If we want to both reason about the sub-array and mutate the original array, we should use splice.

In the following example, we see another difference between the two methods when they are not passed any arguments.

var sumSlice = function(array){
  var copy = array.slice(), total = 0;
  for (var i = 0; i < copy.length; i++){
    total += copy[i];
    }
    return "The sum of " + array + " is: " + total;
}

console.log(sumSlice([1,2,3,4])); //The sum of 1,2,3,4 is: 10

var sumSplice = function(array){
  var copy = array.splice(), total = 0;
  for (var i = 0; i < copy.length; i++) {
    total += copy[i];
  }
  return "The sum of " + array + " is: " + total;
};

console.log(sumSplice([1,2,3,4]); //The sum of 1,2,3,4 is: 0

This happens because when we pass no arguments to slice we are given a copy of the entire array. In this case we are able to reason about copy within the for loop in sum the same as the input array.

However, check out what happens in sumSplice. Because we haven't passed in any arguments to splice, there have been no elements removed from array and passed into copy. We are iterating over the elements of an empty array and passing the value of each element to total. Since there are no elements in copy, total remains at the value we assigned it to, 0.
