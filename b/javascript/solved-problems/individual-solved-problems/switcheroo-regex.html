This problem helped me understand the Regular Expressions function .replace.

Codewars challenge title: Switcheroo by petegarvin1. We are supposed to switch the 'a' characters to 'b' characters and vice versa. The input will be a string.

function switcheroo(x) {
    
}

//test cases:

switcheroo('acb') --> 'bca'
switcheroo('aabacbaa') --> 'bbabcabb'

My solution (no RegExp, though I am proud of using the array method .map on the input string here)

function switcheroo(x){
  return [].map.call(x, function(item) {
    if (item === 'b') {
      return 'a'
    } else if (item === 'a') {
      return 'b'; 
    } else {
      return item
    }
  }).join('');
};

However, let's check out a shorter solution provided by ScratchFandango

function switcheroo(x){
  return x.replace(/[ab]/g, (x) => (x === 'a')? 'b':'a')
}

The author uses .replace to search though the string x and replace all 'a' characters with 'b' characters and vide versa. .replace takes two parameters, the first being what we're looking to replace and then the second is what we replace it with.

Note that the first parameter can be either a RegExp object or a string literal. The second parameter can either be a string literal which will denote verbatim what we want to replace the string from the first parameter with. Alternatively, we can pass in a function as the second parameter, which will create the new substring.

In the case above, the author uses a RegExp object for the first argument to replace: /[ab]/g which will search the input string for a or b characters. If either of those characters is found, then it the matching character will be replaced by the work done by the function (x) => (x === 'a')? 'b':'a'

Important to note that the callback to replace is only invoked when a string that the first argument describes is found in the string x. Thus, the character 'c' is unaffected.
