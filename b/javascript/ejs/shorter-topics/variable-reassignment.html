In JavaScript there are certain data types that are treated differently. For example, variables that point to primitive data types that make references to each other will not affect each other's values.

var pete = 'pete';
var name = pete;
pete = 'happy';

console.log(pete, name); 
// 'happy' 'pete'

Above we've seen that name makes a reference the value of pete upon definition. In a subsequent line pete is reassigned to the string 'happy'. The value of name is unchanged. This is because name was assigned to the value pointed to by pete when name was first declared. name doesn't care what happens with pete after name has been declared.

var myObj = {name:'myObj'};
var yourObj = myObj;

console.log(myObj);
// {name;'myObj'}

yourObj.age = 28;

console.log(myObj);
// {name:'myObj', age:28}

Above, we can see that both myObj and yourObj make reference to the same object and can both thus affect the object that both variable names point to.

This rule can describe why sometimes we see variables that point to objects, arrays, or functions being changed unexpectedly in our program. Alternatively, variables that point to primitives will only be affected by assignment to their own name.
