Chapter two describes program structure and how the interpreter reads a program. The interpreter will evaluate a program top to bottom and there are various ways to make sure that only code that is desired is evaluated. A real world example of this could be that our Facebook newsfeed is only populated by posts of pages that a user follows/is friends with. Even though posts exist that are from pages not linked to a user, a Facebook profile page needs to instruct the browser to skip those non-linked posts when constructing a newsfeed.

Let's check out some of the examples of looping control flow that Haverbeke touches on, using a program to calculate 3^8 without using Math.Pow()

while

var counter = 0, result = 1;
while (counter > 8) {
  result *= 3;
  counter++;
} 
console.log(result); //6561
console.log(result === (Math.pow(3,8))); // true

The while loop consists of the while keyword followed by a conditional expression found placed within (). If the condition holds, than the succeeding blocks will be evaluated. Otherwise the lines of code that comes after the block will be evaluated. In order to calculate 3 ^ 8, we declare result value, set to 1, and then multiply it by 3 for as many times as the exponent requires (8 times in this case). counter variable starts at 0 and will be incremented by one for each iteration until counter === 8, in which case the conditional will not hold and we have a completed result variable.

do

var result = 1;
do {
    result *= 3;    
} while (result !== Math.pow(3,8));
console.log(result); // 6561

do is similar to while, and even includes the while keyword in its syntax. A crucial difference is that the do loop will execute once BEFORE evaluating the conditional expression following while. This means that result will be reassigned to 3 before checking any conditionals.

The example in the text makes more sense for a do loop.

do {
    var name = prompt("What's your name?");
} while (!name);
console.log(name);

The programcito here prompts the user to provide a name and will only ask again if an empty string is provided as name. Here the program only loops if it needs to.

for

Haverbeke describes the rise of the for on page 33: "Many loops follow the pattern (seen before). First, a "counter" variable is created to track the progress of the loop. Then comes a while loop, whose test expression usually checks whether the counter has reached some boundary yet. At the end of the loop body, the counter is updated to track progress."

"(The following program using for is exactly equivalent to the earlier (examples). The only change is that all the statements that are related to the 'state' of the loop are now grouped together".

This means that the counter variable has been moved to be an essential part of the for syntax and doesn't require an external variable. This is a cool example of the trend in programming to abstract away details when a process is very similar and could be done in a more efficient way.

var result = 1;
for (var i = 0; i < 8; i++) {
  result *= 3;  
}
console.log(result);
console.log(result === (Math.pow(3,8)));

We've seen three different looping control flow mechanisms to utilize the power of a computer to repeat code with updated state: while,do, and for.
