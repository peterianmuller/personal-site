After finishing my write-up of loops I thought that it would be cool to try and implement the native Math.pow() function using loops.

The result:

power = (base, exponent) => {
  if (exponent === 0 || base === 1) {
    return 1;
  }
  var result = 1;
  if (exponent > 0) {
    for (var i = 0; i < exponent; i++) {
      result *= base;
    }  
  } else if (exponent < 0) {
      for (var i = 0; i > exponent; i--) {
        result *= base;
    }
    result = 1 / result;
  }
  return result; 
};


console.log(power(5,-8) === Math.pow(5,-8)); //true
console.log(power(2,3) === Math.pow(2,3));  //true
console.log(power(0,1) === Math.pow(0,1));  //true
console.log(power(0,0) === Math.pow(0,0));  //true
console.log(power(0,6) === Math.pow(0,6));  //true
console.log(power(-2,3) === Math.pow(-2,3)); //true
console.log(power(0,-3) === Math.pow(0,-3)); //true

I have tried to deal with a few simple rules of exponentiation:

  1. 0 to the power of anything is 0

  2. 1 to the power of anything is 1

  3. x ^ y can be simplified to y occurrences of x multiplied with itself:

    2 ^ 3;

    2 * 2 * 2;

    8;

    Within the function, once we've established that we are dealing with a base and exponent > 0 I declare a result variable and set it to 1. I then multiple that result variable by the base a number of times equal to the exponent.

  4. x ^ -y y occurrences of (1/x) multiplied with (1/x):

    2 ^ -3;

    1/2 * 1/2 * 1/2;

    1/8;

    The result variable does the same function for the negative exponents as it does for the positive exponent, although once I have a rsult variable equal to the base multiplied by itself equal amount of times to the exponent I reassign result to be 1 over itself.

What is cool here is that we can think of exponentiation as a form of looping. All a positive exponent really does is tell us how many times we want to multiply a base with itself. This program is a cool example of how computer programs come from "the study of knowledge from an imperative point of view as opposed to a declarative point of view" (Abelson et al, xxiii). This 'imperative point of view' means that we can see how a number raised to an exponent is calculated, not just what the product of that calculation is.
