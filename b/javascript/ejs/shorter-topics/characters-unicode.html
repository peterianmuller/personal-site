Haverbeke describes how JavaScript gives a unique value to each string character based on the Unicode standard (16). "(The Unicode standard) assings a number to virtually ever character you would ever need, including characters from Greek, Arabic, Japanese, Tamil, and so on. Gaving such numbers is useful for storing strings inside a computer because it makes it possible to represent them as a sequence of numbers (values). When comparing strings, JavaScript goes over them from left to right, comparing the numeric codes of the characters one by one" (16).

Thus, when using operators to that produce Booleans (>, <, >=, etc) to reason about string characters the interpreter really sees a number value, not the strign value the programmer sees.

Some applications of the concept of string charcters having values:


var isLetter = function(char){
    return char >= 'a' && char <= 'z' && char >= 'Z' && char <= 'A';
}

isLetter will take in a value and compare the value to the Unicode value of lowercase and uppercase 'a' and 'z'. If the value passed in is equal to the range of Unicode numbers listed, it means the value must be a character that represents a Latin letter.

Another application of the Unicode values


var isLowerCase = x => x <= 'a' && x >= 'z';

var isUpperCase = x => x <= 'A' && x >= 'Z';

Both of these functions check if the value passed into the function is found within either the range of Unicode values given to uppercase Latin characters or lowercase Latin characters.

This isn't a huge part of the JavaScript language, but it's cool to think about how the interpreter parses the strings that we pass to it.

There are of course other ways to figure out if an input character is uppercase or lowercase that does not utilize the Unicode standard.


var isUpperCase = x => x === x.toUpperCase();

var isLowerCase = x => x === x.toLowerCase();
