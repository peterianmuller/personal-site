Expressions and Statements

An expressions is a "Fragment of code that produces a value" (23). Some examples of expressions:

Values written out:

22, "cat"

values between (), such as in the loop structures we'll see soon: 

for (4 === 4)

Binary operators applied to two values/ urinary values applied to one value

Because Javascript is a "language-based interface", we can combine expressions to make more complicated computations (23). "Expressions can nest in a way very similar to the way subsentences in human languages are nested - a subsentence can contain its own subsentences and so on,"(23). In javascript we can combine expressions to express more complex operations.

Let's see how we can go from using the functionality of the basic * operator to abstract out slightly more complex procedure.

3 * 3;
// 9 (basic expression that denotes multiplication)

var square = function(n) {
  return n * n;
};

square(9);

// 81  (we've combined a function type with the * operator to create a function that does a more complex operation)

var areaRightTri = function(base, height) {
  return (base * height) / 2;
}

areaRightTri(3,4);

// 12 here we've used expressions including the * operator and the / operator to create the more complex idea of calculating the area of a triangle.

Haverbeke uses the analogy of an expression as a sentence fragment and as a statement as a full sentence. (24 top)

If we place a semi-colon after an expression it becomes an "expression statement", which is a vary basic type of statement.

7
// expression
7;
// expression statement

Note there are some expressions that can't be turned into expressions statements.

(4 > 5;)
// SyntaxError: Unexpected token ;
3 > 6; && true;
// SyntaxError: Unexpected token &&

An expression is useful when it produces a value. However, a statement that simply produces a value, such as the expression statements above, don't really do anything other than create the value named in storage for the duration of the program.

When a statement does something other than simply create a value, it is said to have "side effects" - anything more than simply naming a value to memory for a second (24).

Examples of statements that have don't have side effects vs those that do have side effects:

7;
// 7 (no side effect, simply creates the value in memory for runtime).
console.log(7);
// 7 (side effect of printing the value 7 to a console).

var catName = "bill";
// "bill".  variable declaration statement - affects the outside world b/c value is saved to hardware memory and isn't just thrown away

Variables

So we've learned that we can create a value simply by typing it's name:

true;
false;
NaN;
undefined;

We can also create a value by typing the data we desire that will be saved as a specific type:

5;
"cat";
"";
19e3;

"To catch and hold values, Javascript provides a thing called a variable,"(24). Note the use of "catch and hold" here and not "a variable is equal to" as variables often are in mathematics. Variables in Javascript "point" to values , "They do not contain them, they grasp them - two variables can refer to the same value" (25).

We can create variables using another type of statement, a "variable declaration statement", which involves using the keyword var followed by the name of the variable followed by the assignment operator = followed by the value that the vairable points to

var pointsToANumber = 17;

Note that we can set, or "initialize" a variable to a compound expression. all variables have to be resolved ot value on the right side of the assignment operator, so in this case our variable will be the result of the complex operator described:

var complexNumberOrNah = 4 * 3 + 2;
complexNumberOrNah;
// 14

Variable names can contain alphanumeric characters, must start with a number, but can contain $ and _ (25).

Importante! "When a variable points at a value, that does not mean it is tied to that value forever"(25). We can use the = at any time to reassign the variable to point at a different value.

var weather = "sunny";
weather = "cloudy";
weather;
// "cloudy"
weather = "snowy";
weather;
// "snowy"

EJS has a cool example of updating variables and debt by a friend who is really tryna get it together, Luigi:

var luigisDebt = 140;
luigisDebt = luigisDebt - 35;
// Above the expression to the right of the = must be resolved to a value, so luigisDebt is converted to 140, the value it points to at that point.
luigisDebt;
// 105

All variables that are initialized but not given a value point to "undefined". Also, we can declare multiple variables using one =

var month = "January", day = "Monday", date = "31st";
"The month is " + month + ". The day is " + day + ", the " + date;
// "The month is January. The day is Monday, the 31st"

While when we open our text editor it seems like a black page, but if we are programming in the browser environment there are hella variables already declared: such as console.log(), prompt(), confirm(), and a host of DOM stuff that is too long to list.

Functions

The values I've listed above are of type function. "A function is a piece of program wrapped in a value" (27). Functions can be thought of as mini (or sometimes not so mini) programs in themselves. When we use functions, this is called "invoking, calling, or applying" it. For example, when we apply the alert function, we should see a text box displaying the text we've written into the function:

alert("Good freakin' morning");
// (Box pops up with a dope greeting)

When we invoke, the Values that we provide inside the () after the function name are called arguments (27). In the case above the alert function takes one argument of type string, though functions can take none, or many more and of several different types.

console.log()

The console.log() function is a way to make values passed as arguments visible to a console. Web browsers, Node.js, and several other javascript environments have consoles which can be used as a canvas to paint values passed to console.log().

Note that we said variable names cannot contain punctuation other than $ and _. However, console.log "is actually an expression that retrieves the log property from the value held by the console value." Thus, console.log() is more of a property access than a function per se, hence the inclusion of . in the variable name (27).

Return Values

Functions can do either or both of the following actions:

Provide side effects (changing the world outside that function in some way. We've seen that the console.log function prints a value to a console. Alert pops up a text box. However, these functions technically carry no value (other than the default undefined))

Return values. Functions can be useful in ways that don't necessarily change the outside world. Some examples of functions that return values but don't have side effects:

Math.min(3,100);
// 3
Math.sqrt(9);
// 3

"When a function produces a value, it is said to return that value. Anything that produces a value is an expression in Javascript, which means that function calls can be used within larger expressions"(28). Thus, functions that return values can be expressions, or the sentence fragments that allow us to build larger and compare complex ideas.

Prompt and confirm

prompt is a built-in function that allows a program to ask the user a question via a pop-up box. There are two arguments, the 1st being the question and the second being the default text shown in the answer field (27).

prompt("Who is your favorite Khardashian?", "Rob?");

confirm is a built-in function that provides a pop-up box with text as the first argument. The user then can choose OK or Cancel in response to the text. Ok gets parsed to boolean true, and Cancel to false.

var theNumber = Number(prompt("Pick a number", ""));
alert("Your number is the cube root of " + theNumber * theNumber * theNumber);

Control Flow

While we have seen that multiple expressions can be nested inside of a statement, a statement is the end of reasoning on the expressions that make up that statement. The type of code we have seen until now, statements are interpreted top to bottom.

Below this program has two statements that will be interpreted in order, from top to bottom.

var theNumber = Number(prompt("Pick a number", ""));
alert("Your number is the cube root of " + theNumber * theNumber * theNumber);

If we save the prompt function to a variable, the value that variable points to is the user's response. Also note that we are using the native function Number() that converts it's parameters to a number type. when a user provides input to prompt, that input is saved as type string so Number() will convert the string to a number (29).

Conditional Execution 1: if/else branches

Typically, we only want to execute code under certain circumstances and not just every statement in order from top to bottom. We can use various constructs to perform this conditional execution.

We use Boolean values to determine if we take a branch set up by our code. Booleans are two-valued data, so in terms of choice they make sense to decide if we want to have our code take a certain action vs. another (29)

In our previous program, we assumed that the user would pass a string that contains numeric characters that would be parsed into type number by the Number function.

what if the user passes "pizza"? We can allow for this case to happen and respond with adequte code. As of yet there is no cube root for "pizza".

var theNumber = Number(prompt("Pick a number", ""));
if (!isNaN(theNumber))
alert("Your number is the cube root of " + theNumber * theNumber * theNumber);

Here, we've used an interesting trick to ensure no non-number input will pass through to the alert. isNaN() is a built in function like alert, prompt, etc. isNan() is passed a value and determines if that value is of type number, if not isNaN will return true. In our program above, we've placed the not operator before the expression of isNaN, so if theNumber is a number, !isNaN() will be true, allowing the program to enter the body of the if statement.

However, we still haven't instructed our code to do something else if a number was not provided. We can use the handy keyword else for that.

var theNumber = Number(prompt("Pick a number", ""));
if (!isNaN(theNumber))
  alert("Your number is the cube root of " + theNumber * theNumber * theNumber);
else
  alert("Hey, why didn't you give me a number!?");

There is a third option for a branch between an if and else branch, which is an else if branch.

var choice = prompt("Pick a number from 1 to 10","...");
if (choice < 10)
  alert("That's a small number");
else if (choice < 100)
  alert("That's a bit bigger");
else
  alert("That's big!!");

Above, the interpreter will receive an input for choice and than find the corresponding branch that has an expression in parenthesis that evaluates to true, starting with the if statement.

Conditional Execution 2: Looping constructs

while loops

We've looked at how a program can skip steps and dispatch on or choose a certain option based on a condition. However, what if we want a program to repeat a step over and over. I mean one of the power of a computer is the ability to compute operations with speed, so if we want the program to do something over and over again, there must be a more efficient way to do that other than the following. Let's write out a program that prints out all even numbers from 0 to 12

console.log(0);
      console.log(2);
      console.log(4);
      console.log(6);
      console.log(8);
      console.log(10);
      console.log(12);

"That works, but the idea of writing a program is to make something less work, not more" (31). Haverbeke bring up the point here that expanding this program to print out the numbers up to 1000 would be ridic, and it's true! Looping constructs are one the most basic examples of the powerful advantage a computer has: to be able to do many tasks super quickly. "Looping control flow allows us to go back to some point in the program where we where before and repeat it with our current program state," (31). Importantly here the state of the program changes according to the rules we set. Thus we don't have to manually write out each number for the program to instruct the computer to print. #automation at its finest.

var number = 0;
      while (number <= 12) {
        console.log(number);
        number = number + 2;
      }
      // 0
      // 2

When we start a statement with while, we are creating a loop. The expression inside the parenthesis that follows "while" checks if the following statement should be executed. If the expression in the () evaluates to true, the subsequent code will get executed.

In the code above: 1) Interpreter encounters a variable declaration statement for number that points to 0. 2) Encounters while, so the interpreter knows things are about to get loopy. 3) Checks the expression in (). 4) If true, the program prints the current state of the variable number to a console and adds 2 to the variable number. 5) At this point the magic happens, and the code will return to the line with the expression in () and see if that expression still evaluates to true with the current state of the variable number. The program will then execute the statement following the loop if this expression evaluates to true. 6) Once the expression in the () evaluates to false, the program breaks out of the loop and moves to the line after the closing bracket of the loop.

Also good to know - curly braces group statements together to count as a single statement. Just like multiple expressions inside a () count as a single expression (31).

What's interesting about the loop above is that the same number that we want to print also keeps track of when we want to jump out of the loop. But many times, programs reason about numbers/data different than the number of times a program is supposed to repeat itself. For example let's use a loop to calculate 2^10:

var result = 1;
      var counter = 0;
      while(counter < 10) {
        result *= 2;
        counter += 1;
      }
      result;
      // 1024

Here we've introduced the idea of a separate counter variable. I like to think of parallel processes going on here: the counting of repetitions, or "iterations" of the program and the work being done to achieve the original goal of the program. Note that our original goal of calculating 2^10 should not require us to think about a counter variable. Most all calculators have eliminated the need to tell the computer both the desired calculation (2^10) and the necessary iterations (10). We'll see later how we can hide, or abstract away, the need for a counter variable.

do loops

do loops are similar to while loops, but they perform the statement that follows the conditional statement at least once.

do {
        var name = prompt("Who are you?");
      } while (!name);
      console.log(name);

This is syntatically interesting. We're telling the interpreter to "do(repeat) the prompt function while there is no valid name" "...The ! operator will convert a value to Boolean type before negating it, and all string except "" convert to true" (32). Thus the while statement here will first convert the user input to a Boolean. The user input will be true if there is a non-empty string entered. Then the ! is applied and the expression with the valid string, which is true, will be negated to false. We know that conditional expressions that are false break out of loops, so a valid string's truthiness negated to falsiness allows the program to break out of the loop.

Consider what happens when we omit the !:

do {
        var name = prompt("Who are you?");
      } while (name);
      console.log(name);

Here, the conditional expression will evaluate to true if there is a valid string entered, which will lead to repeated prompt functions. The only way to get the conditional expression to evaluate to false is to enter an empty string.
However, we want the user to enter a name, not an empty string.

The do loop seems make sense for situations where looping might not be necessary if the code executes as desired the first time it is run. Alternatively, the while loop needs the looping behavoir to successfully execute the code. There's no way to print out 2^10 using iteration without a loop repeating itself.

for Loops

There is a common pattern of loops that require a counter variable to keep track of hoe many times a loop needs to repeat until the work of a program is done. Anytime prorgrammers find that a behavoir is repeated many times, there is an effort to create a construct to make that behavoir more easy for programmer to perform - Abstraction!

the for Loop, in this case the for-in loop, has a way to biuld in a counter variable into the body of the loop so it becomes unnecessary (and messy) to declare a counter variable in the code before the loop.

for (var number = 0; number <= 12; number += 2)
        console.log(number);
      // 0
      // 2
      // etc.

The conditional expression that follows the for keyword contains two ;. The first expression declares the coutner variable. The second expression (after the first ;) checks to see if the loop should repeat or if the code should break out of the code. This second expression is what would be found in the () after the while loop. the third expression updates the "state of the loop after each iteration" (33). Note that the updating happens after the work of the loop is done.

2^10 with for loop:

var result = 1;
      for (var i = 0; i < 10; i++) {
        result *= 2;
      }
      console.log(result);

the break statement

So we know a few ways to break out of a loop. - while (false) - When second expression in () for a for loop evaluates to false.

However there is a way to break out of a loop without filling in the second expression in the () after the for keyword.

Let's say we want to find the first number divisible by 7 starting from 20 and increasing by 1. Lol this is like the coding version of Spanish textbooks teaching you how to say "I'd like three mangos, Juan".

for (var current = 20; ; current++) {
        if (current % 7 === 0)
        break;
      }
      console.log(current);
      // 21

      //Let's say we want to find the first number divisible by 4 and 7 starting from 20 and counting up by one.

      for (var current = 20; ; current++) {
        if (current % 4 === 0 && current % 7 === 0)
        break;
      }
      console.log(current);
      // 28

The break keyword makes sense here because we are trying to find the number that would go in that second conditional expression.

Dispatching on a Value with Switch AKA why my Javascript is Lisp mantra breaks down :(

An alternative to multiple if/else statements is the wack ass switch statement, which allows us to solve a "dispatch" or list of options for our code to choose from.

// example of (somewhat complicated) dispatch:
      if (variable === "value 1") action1();
      else if (variable === "value 2") action 2();
      else if (variable === "value 3") action 3();
      else defaultAction();

      // solving a dispatch with switch!

      switch (prompt("How deep is the puddle you stepped in today on the way to the mini-mart to buy a doughnut cuz Macklemore said he ate a doughnut recently?")) {
        case "I did not step in a puddle":
          console.log("Way to go!");
        case "shallow":
          console.log("You're lucky");
          break;
        case "medium":
          console.log("Kinda lucky, kinda not");
          break;
        default:
          console.log("Damn man! Should've bought a moped");
          break;
      }

Once the input is provided, the code will search for the case that matches the input and then execute the code that follows the matching case. The interpreter will jump out of the switch statement once it encounters a break statement (35). Note that not every case needs a break statement. In the code above if the user did not step into a puddle, two messages will be printed to the console: "Way to go!" and "You're lucky"!

Summary!

Programs are made of statements that are themselves made of statements. Remember curly braces can wrap several statements into one block and will be counted as a single statement. Statements are built of expressions. (37).

When we place statements one after another, they are executed as such. We have seen two methods to disrupt the flow of control: - conditional statements: if else switch - looping statements: for while do

Variables allow the program to save values to memory and to "track state" in a program. Variables and the values they point to are called "the environment". JavaScript comes with built in variables, such as console.log(), typeof(), etc.

We looked at some basic functions. Functions are programs wrapped in values. They manipulate data but are data themselves!!! Doughnuts that make more doughnuts. Functions can be invoked, called, or applied by writing the following:

functionName(arguments);

Since the above function invocation ends in a ; it is an expression. Function expressions may produce values, side effects, or both (37).
