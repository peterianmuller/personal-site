Moving Around

"Absolute path" is a filepath to locate a file that is the same no matter where you are in the terminal. The way to describe an absolute path is to start at the root directory. For example, to enter the Documents directory via absoluta path, we can type in:

cd /Users/petermuller/Documents

We would use a diferent notation if we were already in the Desktop directory and wanted to move forward. "Moving forward" means we need to use ./ in our bash script. Let's make sure we're at the root directory by using an absolute path to get there:

cd /Users/petermuller

Now to move forward from the root directory we need to use our homie ./

cd ./Desktop

Now we're in the Desktop directory!

If we want to see the filepath for a nested folder we can drag said folder to the terminal window and that will show us what's poppin'.

Peters-MacBook-Pro:Downloads petermuller$ /Users/petermuller/Desktop/Personal/Original\ Music/DJ\ Library/Hip\ Hop/Soulquarians/D'angelo

Copying

Copying uses the cp command. The way to use cp to copy a file into the current directory is:

cp TestFile.js TestFile-Copy.txt

AKA

cp originalfilename.extenseion copyfilename.extension

If we want to copy a file and have that copy appear in a different directory we can do:

cp originalfilename.extension /Some/Folder/

Let's make a copy of our copy that we created in the Downloads directory

cp TestFile-Copy.js /Users/petermuller/Downloads

Moving

mv can be used to 1) move a file from one directory to another and 2) rename a file

  1. Using mv to move a file to a new directory

First let's get ourselves into our test directory:

cd /Users/petermuller/Desktop/test

Now let's use mv to move our test file to another directory mv Testfile.js /Users/petermuller/Music

Now Testfile.js has been moved from the test directory and the file resides in the Music directory, whose absolute path is: /Users/petermuller/Music

The algorithm for moving a file using the mv looks like

mv filename.extension /Some/Folder

  1. Using mv to rename a file.

Enter directory where target file is. In our case, I placed TestFile-Copy.js back in the test directory below the Dektop directory.

cd /Users/petermuller/Desktop/test

mv TestFile-Copy.js TestRenamed.js

ls // TestRenamed.js

mv rename algorithm

mv OriginalFilename.extension NewFileName.extension

Deleting directories

If you type "rm -rf filename.extnsion" this means to delete filename.extension 'forcefull' or 'recursively', which means there will be no stopping the action.
