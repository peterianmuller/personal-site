So the idea of giving the body a border is kinda trippy. Since all html cotnent elements are descendents of the body, if we give the body element a border via CSS, it will go around the entirety of all visible html.

Let's say we have the following html file and we've linked our css in the head tag.

The css will create a border of width 5px around the edge of the page. Note that there is a default margin given so the current version of the margin does not extend all the way to the edge of the screen. Current page.

<!-- main.css -->
body {
  border: 5px solid blue;
}

If we wanted to extend the border to the end of the screen we would include the following in the CSS rule. Remember that only pixel values of 0 do not require a "px" suffix after the number. Border Extended.

<!-- main.css -->
body {
  border: 5px solid blue;
  margin: 0;
}

Centering an html element can be used with the margin: auto; property. However, when applied to our body rules, the border doesn't move. At first I thought: how come the border doesn't move to the center of the page? Then I realized the body is in the center of the page by default. If we wish to move the body's border to the center of the page, then we need to add space on either side of the body. This can be done with the content-width or just width property. If we want the border to take up half the page and respond to a user's change of the size of the screen, we can use the % metric with the width rule. Current Size.

body {
    border: 5px solid blue;
    width: 50%;
    margin: 0 auto;
}
