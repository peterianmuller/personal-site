Using :nth-of-type in a table element

I was playing around with :nth-of-type as part of Estelle Way's great Css Mastery class, and wanted to exapnd on Estelle's excercise of applying styles to rows and columns of a table.

However, what if I wanted to apply different styles to the entire 5th row. I first tried to simply add some new styles that select that row only.

Current Styles:

table.fifth-row-partially-selected tr:nth-of-type(odd) td:nth-of-type(odd),
table.fifth-row-partially-selected tr:nth-of-type(even) td:nth-of-type(even) {
background-color: #900;
color: #fff;
}
table.fifth-row-partially-selected tr:nth-of-type(5) td {
background: linear-gradient(to right, #aaa111, #ddcc11);
color: green;
}

Let's see why the first rules take precedence over the second set of rules. The way I'm targeting td elements in the first set of rules has a specificty of: 0 - 3 - 3. One class, two psudo-selectors (these both have the same specificity), and 3 elements. In the second list of selectors, I'm selecting one class, one psuedo selector, and 3 elements: 0 - 2 - 3.

A cool thing we can do to correctly target all the td elements in our 5th row is to use the :not(selector) psudo selector.

Current Styles:

table.fifth-row-fully-selected tr:nth-of-type(odd):not(:nth-of-type(5)) td:nth-of-type(odd),
table.fifth-row-fully-selected tr:nth-of-type(even) td:nth-of-type(even) {
background-color: #900 ;
color: #fff;
}
table.fifth-row-fully-selected tr:nth-of-type(5) td {
background: linear-gradient(to right, #aaa111, #ddcc11);
color: green;
}
